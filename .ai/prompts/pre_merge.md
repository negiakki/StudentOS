# StudentOS Pre-Merge Review

The feature implementation is complete.

Do not add new functionality.

Perform a final production readiness review.

Review:

## Backend

- endpoints
- validation
- ownership
- error handling
- API contracts

## Frontend

- loading states
- empty states
- error states
- responsiveness
- navigation
- accessibility

## Integration

Verify backend and frontend agree on:

- endpoint paths
- request bodies
- response models
- enum values
- field names

## Quality

- duplicate logic
- dead code
- unused imports
- TODOs
- temporary debugging code

---

Categorize findings as:

- Critical
- Recommended
- Optional

Implement only Critical issues.

If no Critical issues exist, explicitly state:

"Feature is production-ready and can be committed."
